初始设置

bitcoinj内置了日志和断言。无论是否-ea指定标志，默认情况下都会检查断言。日志由SLF4J库处理。它允许您选择您喜欢使用的日志系统，例如 JDK 日志、Android 日志等。默认情况下，我们使用简单的记录器，它将大多数感兴趣的内容打印到 stderr。您可以通过切换 lib 目录中的 jar 文件来选择新的记录器。

bitcoinj使用 Gradle 作为其构建系统并通过 git 分发。您可以使用源代码/jar 下载，但直接从源代码存储库获取更安全。

要获取代码并安装它，请获取Maven或Gradle，并将其添加到您的路径中。还要确保您已安装 Git。您的 Java IDE 可能也有一些 Maven/Gradle 和 Git 集成，但通过命令行使用它们仍然非常有用。

现在获取最新版本的代码。您可以使用使用 Maven或使用 Gradle页面上的说明- 只需运行那里的命令，您就会获得正确版本的代码（除非此网站本身受到攻击）。这旨在防止镜像或源下载受到攻击 - 因为 git 使用源树哈希工作，如果您以正确的方式获取源哈希，则保证您最终获得正确的代码。

您可以在此处阅读完整的计划。

如果你没有这个依赖可以选择1.15.9这个版本。

基本结构

bitcoinj应用程序使用以下对象：

  • NetworkParameters选择您所在的网络（生产或测试）的实例。
  • 一个Wallet用于存储您的ECKeys 和其他数据的实例。
  • PeerGroup管理网络连接的实例。
  • 一个BlockChain管理使比特币正常运行的共享全局数据结构的实例。
  • BlockStore将区块链数据结构保存在某处（例如磁盘）的实例。
  • WalletEventListener实现，接收钱包事件。

为了简化设置，还有一个WalletAppKit对象可以创建上述对象并将它们连接在一起。虽然您可以手动执行此操作（对于大多数“真实”应用，您都会这样做），但此演示应用展示了如何使用应用套件。

钥匙和地址

比特币交易通常将钱发送到公共椭圆曲线密钥。发送者创建一个包含收件人地址的交易，其中地址是其公钥哈希的编码形式。然后，收件人用自己的私钥签署交易以认领硬币。密钥用类表示ECKey。ECKey可以包含私钥，也可以只包含缺少私钥部分的公钥。请注意，在椭圆曲线密码学中，公钥是从私钥派生出来的，因此知道私钥本质上意味着也知道公钥。这与您可能熟悉的其他一些加密系统（如 RSA）不同。

地址是公钥的文本编码。实际上，它是公钥的 160 位哈希值，带有一个版本字节和一些校验和字节，使用比特币特有的base58编码编码成文本。Base58 旨在避免在书写时可能相互混淆的字母和数字，例如 1 和大写字母 i。

由于地址对密钥所针对的网络进行编码，因此我们需要在此处传入网络参数。第二个参数只是用户提供的字符串。如果无法解析或针对错误的网络，则构造函数将抛出异常。

简单来说：生成私钥，私钥推导公钥，公钥推导地址。地址和公钥无法推导私钥。

钱包应用套件

bitcoinj 由多个层组成，每个层的操作级别都低于上一个层。一个典型的想要发送和接收资金的应用程序至少需要一个BlockChain、一个BlockStore、一个PeerGroup和一个Wallet。所有这些对象都需要相互连接，以便数据正确流动。阅读“事物如何组合在一起”以了解有关数据如何通过基于 bitcoinj 的应用程序流动的更多信息。

为了简化这个过程（这通常相当于样板），我们提供了一个名为的高级包装器WalletAppKit。它将 bitcoinj 配置为简化的付款验证模式（而不是完整验证），这是目前最合适的模式，除非您是专家并希望尝试（不完整、可能有缺陷的）完整模式。它提供了一些简单的属性和钩子，让您可以修改默认配置。

该套件有三个参数 - NetworkParameters（库中的几乎所有 API 都需要这个）、用于存储文件的目录以及作为任何创建文件前缀的可选字符串。如果您在同一目录中有多个不同的 bitcoinj 应用程序，而您希望将它们分开，那么这将非常有用。在这种情况下，文件前缀是“forwarding-service”加上网络名称，如果不是主网络（请参阅上面的代码）。
