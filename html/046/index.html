负载均衡

发表于 2022-08-05 更新于 2023-02-02 分类于 分布式
分布式学习笔记

nginx实现负载均衡(反向代理).

阿里云ECS退出后,jar包无法运行:(中间的jar包换成自己的即可)
nohup java -jar ruoyi-admin.jar &
帮助查看:https://www.cnblogs.com/hongmaju/p/15602590.html

阿里云负载均衡有三种:
传统型负载均衡CLB(原SLB): 支持ECS服务器集群
应用负载均衡ALB: 支持ECS服务器集群和函数集群
网络负载均衡NLB: 暂未体验

函数调用的是函数计算(Function Compute)即Serverless.
编写的函数是java8里面的springboot项目,先打包成jar,再压缩成zip上传.
注意监听端口要和tomcat的设置端口一致,另外测试时请注意请求方式和路径.
唯一遗憾的是即使运行一个hello函数,也需要160MB内存,时间为6ms
(当长时间没有访问时,重新唤醒需要200ms,这是难以忍受的)
在分布式追求更高的效率上,很明显即使python也比java要轻松,因为python解释器没有JDK那么臃肿.
